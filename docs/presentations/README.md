# Presentations

[Marp](https://marp.app/) markdown decks for collaborator briefings and design
reviews. Marp lets us keep slides as plain markdown (diff-friendly, lives next
to the source docs it cites) and render to HTML or PDF on demand.

## Files

- `2026-07-snodas-snow-depletion-curves.slides.md` — collaborator overview of
  the SNODAS → `snarea_curve` workflow: what a snow depletion curve is, why
  PRMS/NHM needs one, and how the Driscoll et al. (2017) empirical method and
  Sexstone et al. (2020) lognormal/CV library shaped the pipeline. References
  figures under `../figures/snarea/{oregon,gfv2}/`.

## Naming convention

Filename pattern: `<YYYY>-<topic>.slides.md`. This deck is method-focused and
spans both the Oregon (validation) and gfv2/CONUS (production) fabrics, so it
carries no project-name suffix. A deck whose embedded figures are tied to one
fabric's geometry would name that fabric explicitly.

## Rendering

Pick whichever fits your environment. The `pixi run -e marp` path is the
recommended choice for HPC operators where no system browser is installed —
chrome-headless-shell is downloaded via puppeteer by `marp-setup`.

### Pixi `marp` feature (recommended for HPC / fresh checkouts)

Two-step install — chromium isn't on conda-forge, so PDF rendering also needs
puppeteer's chrome-headless-shell:

```bash
# Step 1 — pulls nodejs + Linux system libs (libgbm + alsa-lib)
pixi install -e marp

# Step 2 — downloads chrome-headless-shell into ~/.cache/puppeteer (~150 MB)
# Required for --pdf / --pptx / --png / --preview output; HTML + server work
# without it. Re-run safely to upgrade.
pixi run -e marp marp-setup
```

Then render any deck via the `render-deck` task (which delegates to
`scripts/render_deck.py`, a thin wrapper that resolves chrome from the
puppeteer cache and sets `MARP_USER=root` for the sandboxless HPC chrome):

```bash
# Render to PDF
pixi run -e marp render-deck docs/presentations/2026-07-snodas-snow-depletion-curves.slides.md --pdf

# Render to HTML (no chromium required at runtime — fast)
pixi run -e marp render-deck docs/presentations/2026-07-snodas-snow-depletion-curves.slides.md --html

# Live-reload server (open the printed URL — works through an SSH port-forward)
pixi run -e marp render-deck docs/presentations/ --server
```

`--allow-local-files` is added automatically by the wrapper (slides reference
figures in `../figures/snarea/<fabric>/` via relative paths).

### VSCode (interactive, alternative)

Install the **Marp for VS Code** extension. Open any `*.slides.md` file and
click "Open Preview" in the editor toolbar. Export via the command palette:
`Marp: Export slide deck...` → PDF / HTML / PPTX. Works on workstations with
Chrome installed; not viable on headless HPC.

### Command line (one-shot via npx, no pixi)

Workstation-only — requires Node.js + a system Chrome/Edge/Firefox for PDF
output. `--html` works without a browser.

```bash
npx --yes @marp-team/marp-cli docs/presentations/2026-07-snodas-snow-depletion-curves.slides.md \
    --pdf --allow-local-files
```

## Editing conventions

- Each `---` on its own line is a slide break.
- The YAML front-matter at the top of each deck controls theme, paginate,
  size, and inline custom CSS — don't strip it.
- Figures are referenced by relative path from the slide-deck file, e.g.
  `![](../figures/snarea/oregon/cv_family.png)`. Figures are generated by
  `scripts/render_snarea_figures.py`; re-run it and commit the updated PNGs
  when the snarea pipeline outputs change.
